# 宠物领养系统 — 类与接口识别分析文档

> 项目名称：`pet-adoption`  
> 技术栈：Spring MVC 5.3 + MyBatis 3.5 + MySQL  
> 包路径：`com.example`  
> 生成日期：2026-06-08

---

## 1. 项目概述

本系统是一个基于 **SSM（Spring + Spring MVC + MyBatis）** 架构的宠物领养 Web 应用，采用经典分层设计：

| 层次 | 包路径 | 职责 |
|------|--------|------|
| 表现层 | `com.example.controller` | 处理 HTTP 请求，提供页面视图与 REST API |
| 业务层 | `com.example.service` / `service.impl` | 封装业务逻辑，定义服务接口与实现 |
| 持久层 | `com.example.mapper` | MyBatis 数据访问接口 |
| 实体层 | `com.example.entity` | 领域模型 / 数据实体 |
| 横切关注点 | `com.example.filter` / `interceptor` | 跨域过滤、前端路由拦截 |

## 5. 业务层接口与实现（Service）

采用 **接口 + 实现类** 模式，实现类使用 `@Service` 注解，通过 `@Autowired` 注入 Mapper。

### 5.1 UserService / UserServiceImpl

**接口定义：**

| 方法 | 返回类型 | 说明 |
|------|----------|------|
| `login(username, password)` | User | 用户登录（MD5 校验） |
| `register(user)` | User | 用户注册 |
| `findById(id)` | User | 按 ID 查询 |
| `findByUsername(username)` | User | 按用户名查询 |
| `findByEmail(email)` | User | 按邮箱查询 |
| `update(user)` | User | 更新用户信息 |
| `deleteById(id)` | void | 删除用户 |
| `findAll()` | List\<User\> | 查询全部用户 |
| `findByRole(role)` | List\<User\> | 按角色查询 |
| `updateStatus(id, status)` | void | 更新用户状态 |
| `checkUsernameExists(username)` | boolean | 检查用户名是否存在 |
| `checkEmailExists(email)` | boolean | 检查邮箱是否存在 |

**实现要点：**
- 密码使用 MD5 加密（`md5()` 私有方法）
- 注册时默认 `role=user`，`status=1`
- 写操作标注 `@Transactional`

## 7. 横切组件

### 7.1 CorsFilter — 跨域过滤器

| 项目 | 说明 |
|------|------|
| 类型 | 类（实现 `javax.servlet.Filter`） |
| 注解 | `@WebFilter("/*")` |
| 职责 | 处理 CORS 跨域请求，支持 OPTIONS 预检 |

**主要配置：**
- `Access-Control-Allow-Origin`：动态 Origin 或 `*`
- `Access-Control-Allow-Credentials`：`true`
- `Access-Control-Allow-Methods`：GET, POST, PUT, DELETE, OPTIONS

---

### 7.2 FrontendInterceptor — 前端路由拦截器

| 项目 | 说明 |
|------|------|
| 类型 | 类（实现 `HandlerInterceptor`） |
| 注解 | `@Component` |
| 职责 | SPA 单页应用路由：非 API/静态资源请求转发至 `index.html` |

**放行规则：**
- `/api/*` — API 请求
- `/static/*` — 静态资源

## 9. 实体关联关系

```
User (1) ──────< (N) Application    用户可提交多个领养申请
Pet  (1) ──────< (N) Application    一只宠物可收到多个申请
User (1) ──────< (N) Pet            用户可发布多只宠物（userId）
User (1) ──────< (N) Application    管理员审核申请（reviewerId）
```

**业务流程关联：**
1. 用户注册/登录 → `User` + `UserService`
2. 浏览宠物 → `Pet` + `PetService`
3. 提交领养申请 → `Application` + `ApplicationService`
4. 管理员审核 → `ApplicationService.approveApplication()` → 联动更新 `Pet.status=已领养`

## 10. 设计模式与架构特点

| 模式/特点 | 应用位置 |
|-----------|----------|
| **分层架构** | Controller → Service → Mapper → Entity |
| **接口隔离** | Service 层定义接口，Impl 实现，便于测试与扩展 |
| **依赖注入** | Spring `@Autowired` 管理 Bean 依赖 |
| **ORM 映射** | MyBatis Mapper 接口 + XML SQL 映射 |
| **双轨表现层** | 传统 `@Controller`（JSP）+ `@RestController`（JSON API） |
| **会话管理** | `HttpSession` 存储登录用户与 token |
| **事务管理** | `@Transactional` 标注写操作 |
